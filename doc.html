用户守则

Cloud I Wanna 用户须遵守以下规则：

  • 禁止任何方式的游戏作弊，包括但不限于代码修改、变速工具、按键脚本、一键JC
  • 禁止恶意刷分、恶意举报
  • 禁止涉及违规的言论、恶意言论、人身攻击
  • 禁止发布无意义、恶意高难的游戏，游戏内禁止出现违规内容
  • 禁止任何方式的爬取、篡改数据库、修改应用程序

    • 注：违规内容包括但不限于涉及政治、广告、引战、谣言、色情、血腥暴力、猎奇的内容

    应用相关

    应用储存

    Cloud I Wanna 大量使用本地储存以提高加载速度和支持离线功能，具体储存内容如下：

  • 基础储存：代码缓存，字体缓存，应用数据等
  • 游戏储存：游戏列表，游戏数据
  • 图像储存：用户头像，游戏封面图，游戏内贴图、背景图片
  • 音频储存：游戏内音效、背景音乐
  • 其它储存：消息记录，玩家存档，制作项目

    • 当储存占用空间过大时，可在设置中清理应用缓存 (该操作不会影响离线使用)，或者选择删除相应的储存数据 (该操作可能会影响离线使用)

    离线使用

    考虑到可能会发生无法连接到服务器的情况，Cloud I Wanna 支持离线使用。离线情况下，游戏和制作功能都能正常使用，但是只能访问已储存到本地的资源。

    版本更新

    Cloud I Wanna 应用程序的版本更新和内容的更新是分离的。在主页左下角可以看到版本信息：v0.x.x build.xxxxx，前者是应用程序版本，后者是内容版本。内容更新频率很高，会在每次打开 Cloud I Wanna 时自动完成，而应用程序只会在一些底层接口变更时更新。应用程序版本更新时，会提示用户重启完成自动更新；跨大版本更新时，需要用户前往官网手动下载更新。

    关卡分区

    关卡分区是为演示、测试、教程、素材等单房间关卡设计的分区（功能上类似 I Wanna Maker），本质上作为游戏分区的辅助，旨在促进作者与作者、作者与玩家的有效交流。完整游戏的制作通常需要花很多时间和精力，而关卡制作简单快速，适合入门学习和积累经验。在这个分区，你可以练习制作、进行教学、测试功能和想法、发布素材，当然也可以在长流程游戏玩累了后玩点短图换换风格！

    定期清理

    关卡分区带有临时性质和快餐性质，容易出现大量低创内容，因此系统会进行定期清理。当一个关卡一段时间都没有通关数据更新时，清理策略如下：

    评分 ≥ 9.6永久保留
    评分 ≥ 9.090天无更新
    评分 ≥ 8.060天无更新
    评分 ≥ 6.030天无更新
    评分 < 6.015天无更新
    评分 < 3.07天无更新
    无评分15天无更新

    引擎特性

    Cloud I Wanna 游戏引擎使用原生 JavaScript 开发，完全还原常规 I Wanna 操作手感，同时修复了由 GameMaker 引擎引起的bug。具体来说，与常用GMS引擎的细微差别如下：

  • 游戏引擎使用双精度浮点（GMS是单精度浮点），且碰撞检测的舍入规则采用一般的四舍五入（相比之下GMS是银行家舍入），因此利用 0.5align 相关bug的特性在 Cloud I Wanna 中失效

    • 游戏存档

    录像回放

    Cloud I Wanna 提供了方便的游戏联机功能

    联机模式

    普通模式：仅显示玩家

    竞技模式：竞技将强制所有玩家同时从头开始，系统会推送玩家的游戏进度信息

    合作模式：玩家间将共享游戏数据，包括存档位置、全局变量、联机触发器和计数器等。合作模式下不会录像，通关记录也不会上传。

    制作相关

    基本使用

    游戏房间

    游戏房间分为四类：

  • 标题房间：游戏最初的房间，一般用做标题
  • 通关房间：到达此房间，游戏即通关，计时会停止，同时上传通关记录
  • 普通房间：计入游戏流程的房间，会存档、计时，属于游戏流程的应当都使用普通房间
  • 选关房间：不属于游戏流程的房间，可用做游戏开始前的各类房间，不会存档、计时

    • 在使用时应当注意：

  • 在开始游戏(或读档)前不应出现普通房间，这会干扰存档的正常工作
  • 标题房间只是游戏的入口，并不一定要做成标题界面，例如你可以将标题房间做成音量检查，然后在后续的选关房间做标题
  • 选关房间的用途很灵活，可用于读档界面、难度选择、开场动画、游戏ED等

    • 使用操作

    图层

    Cloud I Wanna 游戏引擎共有12个图层，对应深度0~11，图层0在最底层，图层11在最顶层。其中图层7是玩家所在图层；图层11是GUI图层，该层相对于游戏窗口位置不变。需要注意的是，由于游戏运行会执行优化，同一层内物体的摆放顺序并不一定会决定最终绘制的前后。

    性能优化

    Cloud I Wanna 游戏引擎使用原生 JavaScript 开发，非编译的特性使得其运行速度低于编译程序，因此在开发时做了大量优化工作以提高游戏性能。虽然如此，由于游戏制作功能复杂，不正确的操作会极大地降低游戏性能。游戏运行速度和物体数量成反比，耗时最多的部分是碰撞检测与处理，且旋转会进一步降低其运行速度，除此之外，不恰当的特效也会影响游戏性能。这里给出一些性能优化建议：

  • 避免大房间，使用多个小房间来代替，同时减少不必要的物体使用
  • 减少碰撞检测的使用，将碰撞逻辑放在物体数量少的一方
  • 设置条件触发的条件时，碰撞检测作为最后的条件 (如使用“[可见]且[碰撞固体]”而不是“[碰撞固体]且[可见]”)
  • 尽量避免使用碰撞反馈 (如使用路径实现板子的来回移动，而不是利用反弹)
  • 尽量使用固定视野而不是跟随视野
  • 尽量减少物体的旋转
  • 弹幕、粒子请使用经过优化的特定动作，而不是使用普通物体实现
